##### 提交规范
1. 实验性的，或将来要添加的，或正在不稳定开发，随时可能出现bug的特性快照，提交到feature/分支；
2. 添加好的功能，这一功能模块已经没有问题的，但需要等待其他模块完成才能发布为版本的开发快照，提交到develop/分支；
3. 多个功能已经开发好的，可以较为稳定地运行的，马上或立即可以作为版本发布的，提交到主master/分支。
